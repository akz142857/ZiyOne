# 淘宝 / 抖音 / 微信电商平台对接方式与准入要求报告

生成时间：2026-07-02 10:58 CST  
适用范围：面向准备开发自研 ERP / 订单履约 / 商品库存 / 服务市场 SaaS 工具的团队，对比淘宝/天猫、抖音电商、微信小店三类平台的商家自研对接与服务商对接路径。

> 结论先行：如果目标是服务多个商家，优先走“服务商 / 服务市场 / 第三方平台”路线；如果只服务自有店铺，走“商家自研 / 自用型应用”路线更快。三家都要求商家授权后才能访问订单、商品、物流等敏感数据；差异主要在云环境强制程度、审核材料、接口费用和服务上架门槛。

## 1. 快速对比表

## 2. 淘宝 / 天猫对接方式

### 2.1 商家自研对接

淘宝官方“品牌商自研系统对接”方案主要面向天猫旗舰店商家，用于把自有 ERP / 电商后台与淘系能力打通。典型能力包括：

- 商品同步、商品发布、商品上下架；
- SKU / 库存同步；
- 订单同步：主动轮询交易 API 或通过 RDS 订单推送；
- 物流发货：在线发货、线下物流发货；
- 退款/退货查询与处理；
- 发票、客服/工单、淘宝小程序等扩展能力。

官方明确前提：商家需要具备自己的技术人员，并拥有自主研发的 ERP / 电商系统。

**应用创建与授权：**

1. 登录淘宝开放平台，创建“商家后台系统应用”。
2. 申请所需 API 权限。
3. 店铺主账号完成授权，平台返回 `sessionkey` / access token。
4. 商家后台系统授权有效期通常为一年，到期需要重新授权。
5. 品牌集团若有多个天猫/淘宝店铺，需要在控制台做授权报备。

**申请材料：**

- 天猫/淘宝店铺账号；
- 产品说明书；
- 自研软件著作权证书；
- 申请主体与软著主体一致。官方文档提示软著可提供且与企业主体一致时，通常较容易通过。

### 2.2 服务商对接

服务商路线适合为多商家提供 ERP、WMS、OMS、库存、售后、客服、营销等 SaaS 工具。与商家自研相比，服务商需要：

- 申请服务商/工具类应用标签或应用权限；
- 上架到服务市场或通过商家授权接入；
- 按商家授权范围访问订单、商品、物流、售后数据；
- 处理多商家隔离、授权生命周期、欠费、数据安全、服务协议等问题。

第三方对接流程通常是：开发者入驻开放平台 → 申请应用标签/权限 → 创建应用获取 AppKey/AppSecret → 商家通过 OAuth 授权 → 使用 sessionkey 调用 API。

### 2.3 技术要求与限制

**聚石塔：**

- 涉及商家或消费者敏感数据的 API，平台规则通常要求必须从聚石塔 ECS 发起调用。
- 淘宝建议商家整套系统托管在聚石塔云上；至少订单处理代码和接口调用代码部署在聚石塔 ECS，店铺相关数据存储在聚石塔 RDS。
- 购买聚石塔机器的账号要与创建应用的淘宝账号一致，平台会校验应用与机器绑定关系。

**奇门：**

- 聚石塔内程序与商家外部系统传递数据，需要通过奇门接口实现。
- 交易订单信息、消费者姓名、手机、收货地址等敏感数据一般不允许直接拿出聚石塔；必须通过官方允许的奇门场景/规则处理。

**订单隐私：**

- 订单中的收件人姓名、手机号、固定电话、详细地址等会脱敏。
- 平台使用 OAID 支持履约。

### 2.4 费用

淘宝开放平台技术服务费主要包括 API 调用费、数据同步费、旺旺/瓦力消息服务费，以及聚石塔云资源费用。

官方技术服务费规则摘要：

| 费用项 | 标准 |
|---|---:|
| 基础 API | 聚石塔内 0.02 元/百次/AppKey；聚石塔外 0.2 元/百次/AppKey |
| 增值 API | 聚石塔内 0.06 元/百次/AppKey；聚石塔外 0.6 元/百次/AppKey |
| 物流流转信息查询 - 菜鸟接口 | 聚石塔内 0.02 元/百次/AppKey；聚石塔外 0.2 元/百次/AppKey |
| 物流异常 - 菜鸟接口 | 聚石塔内 0.1 元/百次/AppKey；聚石塔外 0.5 元/百次/AppKey |
| 数据同步服务 | 标准 0.12 元/百单；优惠阶梯中 100 万单以内 0.1 元/百单、超过部分 0.04 元/百单 |
| 数据同步独享高级版 | 新增订单量计费 + 79 元/天套餐费 |
| 数据同步独享铂金版 | 新增订单量计费 + 149 元/天套餐费 |
| 数据同步独享旗舰版 | 新增订单量计费 + 299 元/天套餐费 |
| 旺旺聊天消息服务 | 0.01 元/百条 |
| 聚石塔机器 | 按 ECS/RDS/Redis 等实际购买规格和周期计费 |

支付方式包括预充值和支付宝代扣。平台按实际 API 调用量、每日新增订单量等扣费。

### 2.5 淘宝适合的接入策略

- **只做自有店铺 ERP / 履约自动化**：走商家后台系统应用，优先只接商品、库存、订单、物流四件套；退款资金逻辑初期保留在卖家中心人工处理。
- **做多商家 SaaS 工具**：走服务商路线，先明确服务市场类目和应用标签；尽早规划聚石塔部署、奇门数据互通和多租户授权模型。
- **高敏感订单数据场景**：必须把聚石塔、OAID、脱敏、奇门作为架构前置约束，而不是上线后补丁。

## 3. 抖音电商 / 抖店对接方式

### 3.1 商家自研对接

抖店开放平台首页将“自研商家”定义为：为具备自主研发能力的商家提供数据能力支持。可通过 API 实现：

- 商品同步、查询、修改、删除；
- 多渠道/区域库存查询、同步、修改；
- 店铺订单展示、打单、发货、改收货地址；
- 售后、物流、营销等能力。

基本流程为：

1. 注册并认证抖店开放平台开发者；
2. 创建自用型/自研应用；
3. 申请 API 权限；
4. 店铺授权；
5. 开发、联调、上线；
6. 按调用量支付开放平台技术服务费。

### 3.2 服务商对接

抖店开放平台将“软件服务商”定义为：基于开放能力开发订单管理、商品搬家等软件，上架服务市场服务抖店商家。典型服务包括：

- 电商 ERP：供应链、商品、订单、库存、物流、逆向处理；
- 订单管理：订单处理、发货履约、售后处理、多店铺运营；
- 商品管理：一键搬家、上架、内容优化、商品数据分析、关键词优化。

官方首页给出的上架四步：

1. 注册认证：填写信息，入驻开放平台；
2. 创建应用：选择应用类型；
3. 开发测试：基于接口联调；
4. 上架应用：上架服务市场，服务商家。

服务商侧要重点关注：应用类型、类目开放状态、服务市场准入、企业资质、产品资料、审核周期、服务保证金/交易服务费等。部分公开资料显示，抖店软件服务商一般只支持企业入驻，个人和个体户通常无法作为软件服务商入驻；具体仍以开放平台后台当前准入页面为准。

### 3.3 商家入驻资质与费用

抖店官网显示，目前支持三类主体开店：

| 主体类型 | 材料 | 审核/验证 | 费用/权益摘要 |
|---|---|---|---|
| 个体工商户 | 营业执照、经营者身份证、店铺名称 | 平台审核约 1-3 个工作日 | 0 元开店；有营业执照商家享更多新商权益 |
| 企业/公司 | 营业执照、法人身份证、店铺名称、品牌资质等 | 审核约 1-3 个工作日；账户验证约 10 小时 | 0 元开店；有营业执照商家享更多新商权益 |
| 个人身份 | 个人身份证、店铺名称、经营地址 | 真实性认证约 1-3 个工作日 | 年累计销售额低于 10 万元；部分类目不支持；权益较少 |

官方“0 元入驻”规则：

- 部分新商家可不先缴保证金，先进行有限制试运营；
- 试运营条件包括：支付订单 ≤ 200 且支付订单总额 ≤ 10,000 元；一般违规 < 2 次、严重及以上违规 < 1 次，且无需缴纳体验保证金；
- 关联主体存在历史违规或经营风险时，平台可限制 0 元入驻或要求补缴保证金；
- 试运营期间存在提现、结算、营销活动、精选联盟等限制；
- 试运营结束或超过阈值后，需要足额缴纳保证金，否则会受到货款直充、提现限制、商品发布限制、成交限制等管控。

### 3.4 API 技术服务费

抖店开放平台技术服务费规则（2026-06-16 更新）明确：收费对象为所有调用开放平台 API 的开发者，包括服务商和商家。

| API 类型 | 调用来源 | 收费标准 | 计费说明 |
|---|---|---:|---|
| 基础 API | 抖店云内调用 | 0.018 元/百次/AppKey | 按实际调用量计费 |
| 基础 API | 抖店云外调用 | 0.18 元/百次/AppKey | 按实际调用量计费 |
| 增值 API | 抖店云内调用 | 0.05 元/百次/AppKey | 按实际调用量计费 |
| 增值 API | 抖店云外调用 | 0.5 元/百次/AppKey | 按实际调用量计费 |

注意：增值 API 未经开放平台允许，禁止抖店云外调用。

支付与结算：

- 技术服务费采用预充值模式；
- 已支付账单不支持退款，账户余额不支持提现、转账；
- 平台每月 5 号统计上一个自然月应用技术服务费并从余额扣除；
- 可申请发票。

欠费后果：

- 欠费第 1、5 天通知告警；
- 第 10 天起，工具型应用服务新签订购冻结，自用型应用的自研应用和商家授权冻结；
- 第 15 天起限流 20%；第 20 天起限流 50%；第 25 天起限流 100%，数据推送产品停止推送。

### 3.5 抖音适合的接入策略

- **低成本试水商家店铺**：可利用 0 元试运营，但要提前规划订单/GMV 达阈值后的保证金补缴。
- **自研履约系统**：优先部署在抖店云内，降低 API 调用成本并避免增值 API 云外限制。
- **服务商 SaaS**：先从订单管理、商品搬家、库存同步、打单发货等高频刚需切入；上线前核对类目、保证金、服务市场协议和收费接口列表。

## 4. 微信小店对接方式

### 4.1 商家自研对接

微信小店可在微信小店平台管理，也可在微信开发者平台查看基础信息、绑定关系和授权关系。

关键账号与配置：

- 微信小店 AppID：小店唯一标识；
- AppSecret：需在微信小店平台生成/重置；
- API IP 白名单：开启后只有白名单 IP 可调用带 AppSecret 的接口；
- 微信开放平台绑定关系：小店与开放平台账号绑定后，可在开发者平台查看关联账号和授权关系。

绑定规则：

- 开放平台账号需完成开发者资质认证；
- 支持同主体或关联主体账号绑定；
- 一个微信小店只能绑定一个开放平台账号；
- 一个开放平台账号最多绑定 20 个微信小店；
- 不支持绑定异主体的微信小店账号。

### 4.2 服务商对接

微信小店服务市场接入指引规定，服务商入驻微信小店服务市场工具专区需满足：

1. 在小商店服务市场具有良好服务能力；
2. 提供商品管理、订单管理、进销存等相关服务能力；
3. 至少具备 PC 端，建议支持移动端，且两端账号互通；
4. 可提供运营人员，完成服务平台测试、客服、售后工作。

接入流程：

1. 理解微信开放平台第三方平台机制；
2. 前往 `open.weixin.qq.com` 申请第三方平台权限集并全网发布；
3. 登录微信服务商平台，进入企业主页；
4. 配置客服和资金账户：客服未设置会导致服务下架，资金账户未设置无法接收结算；
5. 上架“微信小店工具服务”；
6. 配置第三方平台、服务 URL、移动端、小程序 Path、套餐规格 ID 等；
7. 初次建议设置为测试服务进行调试；
8. 服务上线后，商家购买服务，平台将相关权限集静默授权给第三方平台。

套餐要求：需提供 7 天免费版本。

### 4.3 授权与 API 权限集

微信小店工具服务的交易流程具有明显的“服务市场购买 → 静默授权 → 服务商代调用”特点：

- 用户购买服务后，如果微信小店尚未授权相关权限集给第三方平台，服务平台会静默授权并推送授权变更消息；
- 如果已经授权过，则不会重复推送授权变更消息；
- 服务商可通过拉取已授权账号信息接口补偿丢失的授权消息；
- 用户点击“去使用”时，会跳转到服务商 URL 并带上 `code`，服务商调用登录验证换取订单信息；
- 服务商使用第三方平台代微信小店发起接口代调用；
- 用户退款成功后，服务市场会解除微信小店与第三方平台授权关系。

服务类型与权限集示例：

| 服务类型 | 权限集 |
|---|---|
| 商品服务 | 129、130、131、132 |
| 订单服务 | 129、130、131、132 |
| ERP | 129、130、131、132、138、141 |
| 带货工具 | 129、130、131、132、141、167 |
| 厂商代发 | 129、130、131、132 |
| 会员工具 | 129、130、131、132、167、185、186 |
| 供货商代发 | 192 |

### 4.4 费用与资质说明

本次检索到的微信官方文档未给出类似淘宝/抖店那样统一的“API 按百次调用收费”表。服务商侧更明确的是：

- 必须配置资金账户，否则不能接收资金结算；
- 套餐需提供 7 天免费版本；
- 服务上架需通过服务市场审核；
- 商家侧微信小店保证金、类目资质、交易技术服务费、微信支付费率等会按微信小店平台规则、类目和当前后台公示执行。

第三方资料中常见的微信小店商家资质包括：中国大陆合法注册企业/个体工商户营业执照、法定代表人/经营者身份证、结算账户、品牌授权/商标、特殊类目许可证等；但实际金额和类目要求变化较快，应以微信小店后台与官方成长中心规则为准。

### 4.5 微信适合的接入策略

- **已有小店/小程序资产**：优先同主体绑定开放平台账号，梳理 AppID、AppSecret、IP 白名单、授权关系。
- **服务商工具**：通过第三方平台 + 微信服务商平台上架工具服务；设计时不要只依赖消息推送，必须用拉取接口兜底。
- **ERP / 商品订单工具**：优先申请 ERP 权限集，覆盖商品、订单、进销存、带货/会员等后续扩展。

## 5. 三个平台的共同要求

### 5.1 资质共同点

- 都要求主体真实、合法、材料一致；
- 商家主体通常需要营业执照、法人/经营者身份证、店铺名称、结算账户；
- 品牌商品通常需要商标注册证或品牌授权；
- 特殊类目需要行业许可证，如食品、医疗器械、图书、教育培训等；
- 服务商通常比商家自研要求更高，需要企业主体、产品资料、案例/服务能力、客服售后能力、平台协议签署。

### 5.2 授权共同点

- 商家授权是访问店铺数据的前提；
- 服务商不能绕过商家授权读取订单、商品、物流、售后数据；
- 授权关系会过期、解除或因退款/欠费/违规被冻结；
- 需要建设授权状态监控、重授权提醒、权限变更处理和数据隔离。

### 5.3 数据安全共同点

- 订单、手机号、地址、姓名等均属于高敏感数据；
- 平台会脱敏或要求特定云环境调用；
- 系统需保留最小必要字段，不应沉淀无用敏感数据；
- 需要日志审计、访问控制、加密存储、租户隔离和数据删除机制。

## 6. 推荐落地路线

### 6.1 如果是“自有店铺自动化”

1. 先确定平台与店铺主体：淘宝/天猫、抖店、微信小店是否都已开店并完成认证。
2. 按平台创建自研应用：淘宝商家后台系统、抖店自研应用、微信小店开放平台绑定。
3. 只接最小闭环：商品查询/库存同步/订单拉取/发货回传。
4. 退款、售后、发票先保持人工后台处理，避免资金逻辑错误。
5. 稳定后再加订单推送、售后 API、营销 API、会员 API。

### 6.2 如果是“服务多商家的 SaaS / ERP”

1. 优先选一个平台 MVP：建议从抖店或微信小店起步，审核链路和服务市场路径相对清晰；淘宝需更早规划聚石塔/奇门。
2. 准备企业主体、产品说明、后台截图、服务协议、客服与售后机制。
3. 建设统一多租户授权模型：店铺、平台、AppKey、token、权限集、授权有效期。
4. 设计平台适配层：订单、商品、SKU、库存、物流、售后抽象为统一领域模型。
5. 成本模型前置：按 API 调用量、订单量、云内/云外、服务市场抽佣或保证金测算毛利。
6. 上架前准备测试店铺和沙箱/测试服务，验证购买、授权、登录、订单推送、退款解除授权等全流程。

### 6.3 优先级建议

- **最快验证需求**：抖音电商商家自研或服务商工具，开店和接口体系信息最集中，0 元试运营有利于低成本测试。
- **微信生态闭环**：微信小店适合私域、小程序、视频号/微信支付相关场景，但需要处理开放平台、服务平台、第三方平台三套概念。
- **成熟 ERP / 大商家**：淘宝/天猫价值高，但聚石塔、奇门、数据安全和费用结构更复杂，适合作为第二阶段或已有天猫客户时切入。

## 7. 风险与待确认清单

| 风险/待确认项 | 影响 | 建议 |
|---|---|---|
| 平台规则经常更新 | 费用、类目、保证金、权限可能变化 | 每次正式入驻前，以后台准入页面和最新官方规则为准 |
| 服务商类目可能定向准入 | 产品开发完成也可能无法上架 | 先在服务商后台确认类目开放状态，再投入开发 |
| API 费用低估 | 高频同步会产生持续成本 | 做调用量预算，尽量用推送 + 增量接口，部署到官方云内降低成本 |
| 授权/推送丢失 | 订单漏同步、服务不可用 | 所有平台都设计“推送 + 定时拉取 + 对账”机制 |
| 敏感数据合规 | 违规可能下架、限流、清退 | 最小化存储，敏感字段加密，按平台规则处理脱敏数据 |
| 退款和资金逻辑复杂 | 资金错账、投诉 | 初期不自动处理退款资金流，先做查询和提醒 |
